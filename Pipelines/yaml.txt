YAML Syntax:

YAML means YAML Ain't Markup Language.

It is mainly used for Configuation Files.

In DevOps YAML is used in 
- GitHub Actions
- Azure DevOps Pipelines
- Kubernetes
- Ansible 
................

====================

1. Key-Value Pair

Basic Syntax:
----------------
name: Chaithanya
role: Trainer
technlogy: DevOps

name = Chaithanya
role = Trainer
technlogy = DevOps


2. Indentation:
YAML uses spaces, not tabs.

Example:
student1: 
  name: Manan
  course: Azure DevOps
  duration: 40 hours

student2: 
  name: Rishi
  course: Azure DevOps
  duration: 40 hours  

3. Lists / Arrays:
use hyphen - for list items

Example:
prog_languages:
  - C
  - C++
  - Java
  - Python

tools:
  - GitHub
  - Docker
  - Kubernetes
  - Terraform

branches:
  - main
  - master
  - test

4. Nested Objects:
